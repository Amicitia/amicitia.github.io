Patching an Eboot using patch.yml

Why patch.yml?

A patch.yml file is a document that contains formatted instructions for patching a game. It's what we use to get certain mods like the P5 Mod Loader working in RPCS3 (the PS3 emulator).

EBOOT mods don't work in RPCS3, so it's become more efficient to make mods that are compatible with RPCS3 using the patch.yml format than to make a separate EBOOT for every new modification. Additionally, users may want to enable and disable specific mods instead of using all of them at once.

So are PS3 Custom Firmware users out of luck, then?

Fortunately not, thanks to TGE! Now you can create your own modded EBOOT by using a patch.yml file using his program.
Download the Eboot Patcher
and download the patch.yml to get started.

You will also need your choice of a decrypted Persona 5 EBOOT that works on your custom firmware.

Editing the patch.yml
If you're not using one of the EBOOT files linked in the previous section,
you may need to download RPCS3 and install the game's PKGs as well as a PS3 firmware. Rebug CFW works fine for this purpose.

If you already have one of those eboots, ignore the next step and use the hash below:
CEX CFW: PPU-851615a8ca2438050e69e468c5aeeaa53fa25f22
DEX CFW: PPU-d0b4d4ba47cab3d5d8328ade1af75f0ae4861488
Open patch.yml with Notepad or Notepad++ and scroll to the bottom. Here, you'll need to replace the PPU Hash with your unique hash generated by RPCS3. To get the hash, right click the console and set it to "Trace." Then, start the game and quickly close it. Select all and copy the text in the console to notepad and search for "hash." This is what you'll be using.

Enabling/Disabling Eboot mods

If you don't want a particular mod, you can use # symbols to turn the related lines into comments, which will be ignored when patching the EBOOT; or you can just erase the mod from the patch.yml.

Creating your new EBOOT

Now that you've made your changes to the patch.yml, it's time to let TGE's program work its magic on your EBOOT. Open the Windows command prompt and drag the RPCS3PatchEboot EXE onto it. This will input the path to that file for you. Separate each path with a space and repeat this step for the following files:
"Path to EBOOT" "Path to patch YAML" "path to patched EBOOT output"
Use the image to the right for reference.
Your new EBOOT should have been generated! Now just copy it to your game's USRDIR directory (like described in the mod loader guide) and overwrite your original EBOOT, then start the game.

More fun with this method

Certain mods were made specifically for RPCS3's patch.yml so that you can customize their variables. For instance, try adding songs to the game Using the Randomized Battle Music mod.
